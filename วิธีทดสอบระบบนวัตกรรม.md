# วิธีทดสอบระบบนวัตกรรม YRU (สำหรับผู้ส่งผลงาน)

ตอนนี้ **เปิดรับผลงานแล้ว** (สถานะระบบ: `OPEN`, เปิดรับผลงาน: เปิด) ทดสอบได้ทันทีที่:

https://yru-km.pages.dev/?system=innovation&page=login

## บัญชีทดสอบ (Demo Participant)

ใช้บัญชีนี้เพื่อทดลองส่งผลงานโดยไม่ต้องลงทะเบียนใหม่:

| ฟิลด์ | ค่า |
| --- | --- |
| Username | `demo_participant` |
| รหัสผ่าน | `Demo@2569` |
| อีเมล | demo.participant@yru.ac.th |
| หน่วยงาน | สำนักงานประกันคุณภาพ (ORG-009) |

บัญชีนี้มีผลงานตัวอย่างอยู่แล้ว 1 รายการ ("ผลงานสาธิตระบบนวัตกรรม (Demo)") เพื่อให้เห็นว่าหน้า "ผลงานของฉัน" จะแสดงผลอย่างไรหลังบันทึก

## ขั้นตอนทดสอบการส่งผลงาน

1. เปิด https://yru-km.pages.dev/?system=innovation&page=login
2. กรอก Username/รหัสผ่านของ Demo Participant ด้านบน แล้วกด "เข้าสู่ระบบ"
3. กดปุ่ม "ไปยังพื้นที่ส่งผลงานของฉัน" — **รอ 5-10 วินาที** หน้าจะค่อยโหลดข้อมูลขึ้นมา (ปกติของระบบ ไม่ใช่ค้าง)
4. กรอกฟอร์ม "บันทึกผลงาน": หัวข้อประกวด, ชื่อผลงาน, หน่วยงาน, สรุปผลงาน
5. ช่อง "เจ้าของผลงาน/ผู้รับผิดชอบร่วม" กดปุ่ม **"+ เพิ่มผู้รับผิดชอบ"** เพื่อเพิ่มได้หลายคน (กรอกชื่อ/อีเมล/หน่วยงาน/บทบาทของแต่ละคน)
6. กด "บันทึก/ส่งผลงาน"
7. ผลงานจะไปแสดงที่ "ผลงานของฉัน" ด้านขวา พร้อมปุ่มเปิดรายละเอียด และปุ่ม Word/PDF (export ได้จากหน้ารายละเอียดผลงาน)

## ลงทะเบียนบัญชีใหม่ (ถ้าอยากลองในมุมผู้ใช้จริง)

จากหน้า login กดลิงก์ "ยังไม่มีบัญชี? ลงทะเบียนส่งผลงาน" — ฟอร์มจะให้เลือกหน่วยงานหลัก + หน่วยงานย่อย (ถ้ามี) จากรายชื่อจริงของ มรย.

## ฝั่งผู้ดูแลระบบ (Admin)

เข้า https://yru-km.pages.dev/?system=innovation&page=login ด้วย `admin` / `yruadmin2569` แล้วจะเจอแผง **"ตั้งค่าระบบประกวด"** ที่ปรับได้เอง:

- ปีงบประมาณปัจจุบัน
- สถานะระบบ: `SETUP` (เตรียมระบบ) / `OPEN` (เปิดรับ-ประเมิน) / `CLOSED` (ปิดรับ รอประกาศผล) / `PUBLIC` (ประกาศผลแล้ว เผยแพร่)
- เปิด/ปิดรับผลงาน, เปิด/ปิดให้กรรมการประเมิน, เปิด/ปิดเผยแพร่คลังนวัตกรรมสาธารณะ

นอกจากนี้ยังมีแผงจัดการหน่วยงาน (เพิ่มหน่วยงานย่อยใหม่ได้), มอบหมายกรรมการ/เลขานุการตามประเด็น, เกณฑ์ประเมิน, รอบเวลา และ Timeline สาธารณะ — ทุกแผงอยู่ในหน้าเดียวกันหลัง login

## ข้อสังเกตเรื่องความเร็ว

ทุกครั้งที่กดปุ่มเปลี่ยนหน้า (เช่น login → admin/participant) หน้าจะมีช่วง **1-3 วินาทีแรกเป็นหน้าขาว** ก่อนเนื้อหาเริ่มขึ้น และใช้เวลารวมประมาณ 5-10 วินาทีจนข้อมูลครบ เป็นข้อจำกัดของการซ้อน iframe ของ Google Apps Script เอง ไม่ใช่ระบบค้าง — รอสักครู่ข้อมูลจะขึ้นเองแน่นอน
